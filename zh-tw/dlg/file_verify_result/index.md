# 檔案驗證結果對話方塊

**檔案驗證結果**對話方塊提供有關檔案驗證過程結果的信息。當您選擇**驗證**命令時，會出現此對話方塊。

在**檔案驗證結果**對話方塊中，您會看到以下詳細信息：

1. 驗證後兩個檔案是否符合。
2. 原始檔案的路徑、大小和 SHA256 哈希值。
3. 臨時檔案的路徑、大小和 SHA256 哈希值。

如果檔案不符合，可能是由於以下幾個原因：

- 原始檔案在打開後可能已被變更。只有在檔案未被修改的情況下，您才能運行此命令。然而，巨集或外掛程式可以重設指示檔案已變更的標識。
- 原始檔案可能包含無法轉換為 Unicode 的 NULL 或無效字元。
- 原始檔案可能包含不直接對應到指定編碼中的 Unicode 的字元。例如，在 Shift-JIS 編碼中，某些字元會重複列為「IBM 延伸字元」和「NEC 選定的 IBM 延伸字元」。例如，在 Shift-JIS 檔案中程式碼為 0xFA58 的字元（IBM 延伸字元「㈱」），在 EmEditor 中打開時，其 Unicode 字元程式碼將變為 U+3231。此外，如果以 Shift-JIS 儲存，那么它會被存儲為程式碼為 0x878A 的相同字元（即「NEC 選定的 IBM 延伸字元」）。
- 檔案可能是使用**進階打開**命令打開的，並帶有諸如**在每個指定的位元組數處插入換行**或**確保每個檔案末尾都有換行字元**的選項。
- 硬盤或記憶體可能存在問題。如果您嘗試打開網絡上的檔案，可能是網絡問題。

## 刪除臨時檔案核取方塊

選取此選項將刪除儲存的臨時檔案。
