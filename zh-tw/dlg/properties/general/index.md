# 「一般」頁面

**一般** 頁面讓您能設置與基本操作有關的屬性。

## 「自動換行」下拉清單方塊

指定換行方式。您可以從下列幾個選項中選擇一個。

|     |     |
| --- | --- |
| **不換行** | 不換行。 |
| **指定字元** | 按指定的字元長度 (位元數) 換行。字元的長度可以在 **標準行邊距** 文字方塊或 **引用行邊距** 文字方塊中指定。 |
| **視窗寬度** | 按視窗寬度換行。如果調整了視窗大小，換行的位置也會調整。 |
| **頁面寬度** | 按列印的頁面寬度換行。 |

## 「行列顯示」下拉清單方塊

指定要顯示的行數與列的位置。您可以從下列幾個選項中選擇一個。

|     |     |
| --- | --- |
| **顯示坐標** | 行號和列位置會以被顯示的方式計數。如果換行，那么被換行的位置也會被計算在內。列的位置會在換行處被還原為 1。一個全形的字元計為 2 。這是如同文字處理器一樣的顯示方式。 |
| **邏輯坐標** (全形字元計為 2 列) | 用真正的邏輯行號來計算行號和列位置。行號與列位置不取決于換行方式。一個全形字元被計為 2 列。一個 Tab字元被計為 1 個字元。 |
| **邏輯坐標** (全形字元計為 1 列) | 用真正的邏輯行行號來計算行號和列位置。行號與列位置不取決于換行方式。一個全形字元被計為 1 列。一個 Tab字元被計為 1 個字元。 |
| **邏輯 Tab寬度** (全形字元計為 2 列) | 用真正的邏輯行行號來計算行號和列位置。行號與列位置不取決于換行方式。一個全形字元被計為 2 列。當計數一個 Tab字元時，就當它是用空白代替的。 |

## 「標準行邊距」文字方塊

不是引用字元開頭的用位元數標示的行的長度，當 **按指定字元換行** 被選取時。

## 「引用行邊距」文字方塊

引用字元開頭的用位元數標示的行的長度，當 **按指定字元換行** 被選取時。

## 「引用字元」文字方塊

EmEditor 用於 **引用行邊距** 文字方塊上的引用字元。它同樣也適用於 [**複製引用** 命令](../../../cmd/edit/edit_copy_prefix)。

## 「顯示行號」核取方塊

在視窗的左邊顯示行號。列印時如果要列印行號，您將需要勾選 **列印** 頁面上的 **列印行號** 核取方塊。

## 「顯示尺規」核取方塊

在視窗的頂部顯示尺規。

## 「顯示基底為0的行/欄號」核取方塊

顯示基底為0的行號以及欄號，而不是從1開始。這個選項不會更改巨集或外掛程式行為。

## 「運行輸入法編輯器」核取方塊

當啟動 EmEditor 時，同時運行輸入法編輯器 (IME)。

## 「視窗左邊緣無可用空間」

不在 EmEditor 視窗左邊緣顯示空白當不顯示行號時。

## 「顯示頁碼」核取方塊

顯示頁碼。EmEditor 會繪制分頁線，游標所在頁面的頁碼會被顯示在狀態列中。如果顯示行號，那么行的數目是從每頁的頂端開始計算的。如果沒有可用的印表機的話， 頁數可能不會被顯示。

## 「允許插入控制字元」核取方塊

如果勾選了該核取方塊，您能通過輸入 CTRL + 一個字母鍵來插入一個控制字元，或 CTRL + SHIFT + 一個字母鍵，只要它沒有在其他地方被組態為一個快速鍵。如果沒有勾選該核取方塊，您不能這樣插入控制字元。

## 「在剪貼簿保留換行符號 (CR/LF)」核取方塊

如果勾選了該核取方塊，當進行複製貼上時換行符號 也會被保留。例如，僅 LF 的換行符會被複製並貼上到另一處只有 LF 換行符的地方。

## 「總是貼為ANSI」核取方塊

如果勾選了該核取方塊，貼上到剪貼簿上的文字不會作為 Unicode, 而是會一直作為 [系統預設編碼](../../../glossary/systemdefaultencoding)。這個選項在一些情況下可以用到，比如當您想要從其他應用程式中複製文字，因為這些應用程式無法把文字從 [系統預設編碼](../../../glossary/systemdefaultencoding) 轉換為 Unicode 時。

## 「啟用虛空白」核取方塊

如果勾選了該核取方塊，游標游標可以移動超出每一行的末尾，而當插入一個字元在超出每行末尾處時，空白也會被插入。

## 「 Tab/縮排」按鈕

按這個按鈕顯示 [**Tab/縮排** 對話方塊](indent/index) 來指定 **自動縮排** 設定。
