# 「多檔尋找」對話方塊

這個對話方塊會出現，當你選擇 [**多檔尋找** 命令](../../cmd/search/grep) 時。你可以指定一個要尋找的字串以及其他多檔尋找的參數。

## 「尋找」下拉清單方塊

輸入要搜尋的字串。如果這是一個多行的文字方塊，你可以通過按 CTRL + ENTER 來輸入新的行。當下拉清單打開時，你可以通過按 ALT + DELETE 來清除被選取的歷史條目。

## 「>」按鈕

點擊該按鈕來顯示可用命令清單。

|     |     |
| --- | --- |
| **選取內容或游標位置處的單字** | 當該選項被勾選時，對話方塊會把選取的內容或游標處的單字作為尋找文字方塊中的初始字串。 |
| **選取內容** | 當該選項被勾選時，對話方塊會把選取的內容作為尋找文字方塊中的初始字串。 |
| **游標處的單字** | 當該選項被勾選時，對話方塊會把選取的內容作為尋找文字方塊中的初始字串。 |
| **上次使用的值** | 當該選項被勾選時，對話方塊會把上次尋找的字串作尋找文字方塊中的初始字串。 |
| **固定值** | 當該選項被勾選時，對話方塊會把設為固定值的字串作為尋找文字方塊中的初始字串。 |
| **自動儲存** | 把選項 (選取內容或游標處的單字，選取內容，游標處的單字，上次使用的值，或固定值) 儲存為預設值；當下一次對話方塊顯示時，就會是預設的設定。 |
| **多行** | 讓文字方塊在單一行與多行之間切換。 |
| **使用編輯器字型** | 當該選項被勾選時， **尋找** 下拉清單方塊會使用與編輯器相同的字型。 |
| **從批次清單中選擇** | 從批次清單中選擇要搜尋的字串。 |

該命令還包括可用的逸出序列或規則運算式清單。從清單中選擇一個條目會把選取的條目插入到按鈕旁的文字方塊中。

## 「文件類型」下拉清單方塊

要搜尋的文件類型。可以使用通配符（\\* 以及 ?）。同樣，多個文件類型可以用分號（;）來區分。

## 「大小寫須相符」核取方塊

當搜尋一個字串時，區分大小寫。

## 「尋找子資料夾」核取方塊

檢視 **所在資料夾** 下拉清單方塊中資料夾的子資料夾。

## 「全字拼寫須相符」核取方塊

僅搜尋符合的單字。一個單字被定義為一個以這些字元開頭并結尾的字串: A – Z，a – z，0 – 9，或一條底線。由全形字元包圍的字串被認作是一個單字。當使用規則運算式時，該核取方塊可能不能正常運作。所以,當使用規則運算式時，請用運算式來標明單字界限（\\<，\\>，以及 \\b）。

## 「搜尋完成后關閉該對話方塊」核取方塊

當完成搜尋后，關閉該對話方塊。

## 「（無）」選項按鈕

指定字串應按字面意思符合。

## 「規則運算式」選項按鈕

啟用 [規則運算式](../../howto/search/search_regexp)。

## 「逸出序列」選項按鈕

啟用逸出序列。你能把下列字元作為逸出序列。

|     |     |
| --- | --- |
| \\a | 警告（響鈴） |
| \\b | 退格 |
| \\f | 換頁 |
| \\n | 換行 |
| \\t | 水平 Tab |
| \\v | 垂直 Tab |
| \\\ | 反斜杠 |
| \\oooooo | Unicode 字元在八進位運算式中 |
| \\xhhhh | Unicode 字元在十六進位運算式中 |

空字元（\\0）可能不能使用。一定要用 \\n 而不是 \\r。

## 「數字範圍」選項按鈕

啟用 [數字範圍運算式](../../howto/search/number_range_syntax)。

## 「編碼」下拉清單方塊

選擇編碼。如果你選擇 **已組態的編碼**，與打開的文件副檔名相關聯的編碼會被選取。

## 「全部偵測」核取方塊

從所有可用的代碼頁中統計偵測編碼頁。這個偵測可能會出現失誤，特別當文件很小時。這個選項并不包括 HTML/XML 字元集偵測。

## 「偵測 HTML/XML 字元集」核取方塊

在 HTML 文件中尋找"charset=..."，在 XML 文件中尋找"encoding=..."，或在 Phyton 以及 Ruby 文件中尋找"#coding=..."，然后使用向對應的編碼。如果你打開 HTML，XML，Python，或 Ruby 文件，勾選這個參數并清除其他參數。

## 「偵測 UTF-8」核取方塊

統計偵測 UTF-8。

## 「儲存設定」核取方塊

將編碼相關的選項（ **編碼** 下拉清單方塊，以及 **全部偵測**， **偵測 HTML/XML 字元集**，和 **偵測 UTF-8** 核取方塊）儲存為預設值，以便下次打開此對話方塊時還原這些選項。

## 「輸出為」下拉清單方塊

選擇要如何顯示結果。

|     |     |
| --- | --- |
| **文檔** | 把多檔尋找的結果顯示為一個新的文檔。 |
| **輸出欄** | 把多檔尋找的結果顯示為輸出欄的清單。 |
| **直接打開** | 直接打開包含指定字串的文檔。 |
| **直接打開并篩選** | 直接打開包含指定字串的文檔，并把指定字串設為篩選條件。 |

## 「輸出選項」下拉清單方塊

選擇如何顯示結果。

|     |     |
| --- | --- |
| **顯示檔名，行及符合的行** | 將檔案名稱，行號，以及包含搜索字串的整行作為結果顯示。 |
| **僅顯示檔僅顯示檔名案名** | 只有檔案名稱會被顯示為尋找結果，包含搜索字串的整行將不會被顯示。 |
| **僅顯示符合的行** | 僅將包含搜索字串的整行作為結果顯示。 |
| **僅顯示符合的字串** | 僅將符合的字串作為結果顯示。 |
| **顯示檔名，行及符合的字串** | 會顯示檔案名稱，行號，和符合的字串。 |
| **顯示檔名及符合的字串** | 會顯示檔案名稱和符合的字串。 |

## 「停止如果符合數達到」核取方塊/文字方塊

如果在設定此核取方塊後符合數達到指定值，EmEditor 將停止搜索檔案。

## 「尋找」按鈕

用指定的條件開始搜尋。

## 「抽出」按鈕

通過創建新文檔來抽出與指定字串符合的行。單擊右側的 ▼ 按鈕會顯示一個內容功能表，讓你能訪問 [**抽出選項** 對話方塊](../extract_options/index)。

## 「取代 >>」 按鈕

點擊該按鈕儲存目前的要尋找的字串以及參數，關閉目前的對話方塊，並顯示 [**多檔取代** 對話方塊](../replace_in_files/index)。

## 「關閉」按鈕

點擊該按鈕來關閉該對話方塊。

## 「進階」按鈕

點擊該按鈕顯示 [**進階** 對話方塊](../advanced/index) 來設定進階的選項。

## 「加入批次」按鈕

添加目前的設定到批次清單中。點擊 ▼ 按鈕來顯示功能表。

## ▼ 按鈕

|     |     |
| --- | --- |
| **添加連結的檔案...** | 選擇包含多個由換行符號分隔的尋找字串的檔案。這與**匯入**類似，但連結檔案僅包含尋找字串。連結檔案使用的記憶體也較少，因為該檔案在「尋找」清單方塊中顯示為一個條目。 |

## 「批次處理 >>」按鈕

在 **多檔尋找** 對話方塊與 **批次多檔尋找** 對話方塊之間切換。

下列控件僅在該對話方塊延伸為 **批次多檔尋找** 對話方塊時顯示。

## 「儲存至批次」按鈕

儲存目前的設定到批次清單中。

## 清單方塊

顯示用於批次的尋找字串清單。下列字母縮寫用在清單方塊的條件列中，代表指定的尋找條件。

|     |     |
| --- | --- |
| **C** | 大小寫需符合 |
| **R** | 使用規則運算式 |
| **W** | 只搜索單字 |
| **E** | 僅符合 CSV 中的嵌入式換行 |
| **S** | 區別 CR 和 LF |
| **D** | 規則運算式 "." 符合換行符 |
| **B** | 把 Boost.Regex 作為規則運算式引擎 |
| **O** | 把 Onigmo 作為規則運算式引擎 |

## 「全部啟用/停用」核取方塊

啟用或停用清單上所有的條目。

## 「匯入」按鈕

匯入之前匯出的包含尋找條件的 TSV 檔案。

## 「匯出」按鈕

把已定義的尋找條件匯出到一個檔案中。檔案格式可以是 TSV，JavaScript 或 VBScript 巨集格式。

## 「多項尋找」按鈕

單擊此按鈕可同時搜尋所選項目。

## 「批次尋找」按鈕

單擊此按鈕在多個文檔中尋找與批次清單符合的所選條目。單擊右邊的「▼」按鈕會顯示一個內容功能表，讓你能訪問 [**批次選項** 對話方塊](../batch_options/index)。

## 「<< 批次處理」按鈕

在 **批次尋找** 對話方塊與 **尋找** 對話方塊之間切換。

這個對話方塊能通過拖動對話方塊的右下角來調整對話方塊大小。當啟用多行的文字方塊時，按 CTRL + ENTER 能插入新行。
