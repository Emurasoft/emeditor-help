# 尋找上一個警告/Unicode 字元命令

### 摘要

> 尋找上一個無法轉換為要儲存的編碼的警告或 Unicode 字元。

### 說明

> 通過選取字元，尋找上一個無法轉換為要儲存的編碼的 Unicode 字元。如果不能被轉換的 Unicode 字元沒有亮顯，這個命令會亮顯該字元。亮顯的顏色可以在當前組態屬性對話方塊中 [**顯示** 頁面](../../dlg/properties/display/index) 上的 **指定部分** 清單方塊下的 **亮顯 (8)** 設定。
>
> 要用來儲存檔案的編碼有下面的幾個條件決定:
> 要用來儲存檔案的編碼有下面的幾個條件決定:
>
> 如果有這樣的提示消息 "該文檔包含 Unicode 格式的字元。如果您使用選定的編碼儲存該文檔，那么該文檔中的部分字元將會丟失。要保留 Unicode 信息，請點擊「取消」，選擇「另存新檔」，然后在編碼下拉清單中選擇一個 Unicode 選項。要繼續嗎？" 出現，檔案將會用您指定的編碼儲存當您儲存檔案的時候。
>
> 在您選擇 [**取消 Unicode 亮顯** 命令](erase_unicode_hilite) 之後，用來儲存檔案的編碼就是狀態列中顯示的編碼。

### 運行方法

- 預設功能表: **搜尋**
\> **尋找上一個警告/Unicode 字元**
- [全部命令](../tools/all_commands): **搜尋**
\> **尋找上一個警告/Unicode 字元**
- 工具列: 無
- 狀態列: 無
- 預設捷徑: SHIFT+F9

### 外掛程式命令ID

- EEID\_FIND\_PREV\_UNICODE (4376)

### 巨集

#### \[JavaScript\]

> editor.ExecuteCommandByID(4376);

#### \[VBScript\]

> editor.ExecuteCommandByID 4376
