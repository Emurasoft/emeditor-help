# 고급 탭

고급 탭 은 고급 설정들을 사용자 지정하도록 합니다. 이 옵션들을 변경하는 경우
엠에디터를 재시작해야 합니다.

## 문서 간 프로세스를 공유하지 않기 (탭 비활성화) 체크 박스

기본적으로 모든 엠에디터 창은 하나의 프로세스로 실행되며,
탭이 표시되는지 여부에 관계없이 하나의 프로세스 이상으로는 시스템이 실행되지 않습니다.
시스템의 메모리 사용량을 최소화하기 위해 이 옵션은 선택하지 않는 것이 좋습니다.
하지만, 한 응용 프로그램에 오류가 발생되었을 때 엠에디터의 오류 처리기가 저장되지 않은 문서를 저장하려는 시도를 할지라도 응용 프로그램의 오류가 모든 문서의 종료를 발생시킬 수 있습니다.
이 옵션을 선택하면 각 창이 독립적인 프로세스로 실행되며, 한 문서에 응용 프로그램의 오류가 다른 문서의 종료를 발생하지 않게됩니다.
이 항목이 선택된 경우 탭은 사용 불가능 하며 표시되지 않습니다.

## 메모리 사용량 축소를 위하여 임시 파일을 사용 체크 박스

이 항목이 선택된 경우, 편집되는 동안 파일 내용의 일부분이 임시 파일로 저장됩니다; 이 작업은
시스템 메모리 사용량을 줄입니다. 임시 파일의 사용이 실행 속도를 낮추기 때문에 엠에디터는 파일의 크기가 아래의 지정된 크기보다
큰 경우에만 임시 파일을 사용합니다.

## 임시 파일 사용을 위한 파일 사이즈를 최소화 텍스트 박스

파일 크기가 이곳에서 지정된 크기보다 크고메모리 사용량 축소를 위하여 임시 파일을 사용 체크 박스가 선택된
경우에 임시 파일을 사용합니다. 0으로 지정되면 항상 임시 파일을 사용합니다.

## 원본 파일로부터 수정되지 않은 줄을 읽기 체크 박스

이 항목이 선택된 경우, 원본 파일에서 수정되지 않은 줄을 읽어 파일을 여는 속도가 증가되도록 합니다.

## 원본 파일 잠금 체크 박스

이 항목이 선택된 경우, 임시 파일을 사용하여 파일을 열려고 할 때 원본 파일이 잠기게 됩니다.

## 메모리 사이즈 텍스트 박스

이 숫자로 메모리의 한 큰 청크의 가상 메모리 사이즈를 지정합니다.
엠에디터는 매우 큰 파일을 열려고 할 때 작은 메모리의 할당을 먼저 시도하고 점차적으로 메모리 크기를 지정된 크기로 증가합니다.
엠에디터가 더 많은 메모리를 필요로 할 때 메모리 청크를 이와 같이 여럿으로 세분화 합니다.
지정 가능한 숫자는 16과 27 사이입니다. 메모리 청크의 라인의 실제 최대 수는:

|     |     |
| --- | --- |
| 값 | 메모리 청크의 라인의 최대 수 |
| 16 | 2 ^ 16 (65,536) |
| 20 | 2 ^ 20 (1,048,576) |
| 24 | 2 ^ 24 (16,777,216) |
| 26 | 2 ^ 26 (67,108,864) |

## L1 캐시 사이즈 텍스트 박스

이 숫자는 L1 캐시 크기를 지정합니다. L1 캐시는 라인을 기억하는데 사용되는 임시 메모리입니다.
이 숫자는 기억하는 라인의 수에 해당합니다.

## L2 캐시 사이즈 텍스트 박스

이 숫자는 L2 캐시 크기를 지정합니다. L2 캐시는메모리 사이즈 텍스트 박스에서
지정된 청크 메모리의 수를 지정합니다.

## 비동기적으로 열기를 위하여 파일 사이즈를 최소화 텍스트 박스

파일의 크기가 이곳에서 지정된 크기보다 큰경우 비동기적으로 파일을 엽니다.
0으로 지정되면 파일을 항상 비동기적으로 엽니다.

## 시스템 임시 폴더 사용 체크 박스

시스템 임시 폴더를 사용합니다.

## 임시 폴더 텍스트 박스

시스템 임시 폴더가 사용되지 않는 경우 임시 폴더를 지정합니다.
